a Zone file

$ORIGIN coy.im.     ; designates the start of this zone file in the namespace
$TTL 1h                  ; default expiration time of all resource records without their own TTL value
coy.im.  IN  SOA   ns.coy.im. username.coy.im. ( 2007120710 1d 2h 4w 1h )
coy.im.  IN  NS    ns                    ; ns.coy.im is a nameserver for coy.im
coy.im.  IN  NS    ns.somewhere.example. ; ns.somewhere.example is a backup nameserver for coy.im
coy.im.  IN  MX    10 mail.coy.im.  ; mail.coy.im is the mailserver for coy.im
@             IN  MX    20 mail2.coy.im. ; equivalent to above line, "@" represents zone origin
@             IN  MX    50 mail3              ; equivalent to above line, but using a relative host name
coy.im.  IN  A     192.0.2.1             ; IPv4 address for coy.im
              IN  AAAA  2001:db8:10::1        ; IPv6 address for coy.im
ns            IN  A     192.0.2.2             ; IPv4 address for ns.coy.im
              IN  AAAA  2001:db8:10::2        ; IPv6 address for ns.coy.im
www           IN  CNAME coy.im.          ; www.coy.im is an alias for coy.im
wwwtest       IN  CNAME www                   ; wwwtest.coy.im is another alias for www.coy.im
mail          IN  A     192.0.2.3             ; IPv4 address for mail.coy.im
mail2         IN  A     192.0.2.4             ; IPv4 address for mail2.coy.im
mail3         IN  A     192.0.2.5             ; IPv4 address for mail3.coy.im

about security

DNS spoofing (or DNS cache poisoning)

Attack whereby data is introduced into a Domain Name System (DNS) resolver's cache, causing the name server to return an incorrect IP address, diverting traffic to the attacker's computer (or any other computer).

A story about github DDOS.

about security

IDN homograph attack

The internationalized domain name (IDN) homograph attack, by exploiting the fact that many different characters look alike, (i.e., they are homographs, hence the term for the attack)

similar-looking characters such as Greek Ο, Latin O, and Cyrillic О were not assigned the same code

That's why Punycode exists.

about DNSSEC

Domain Name System Security Extensions

The experimental Extensions of IETF DNS for origin authentication of DNS data, authenticated denial of existence, and data integrity.

DNSSEC does not provide confidentiality of data; in particular, all DNSSEC responses are authenticated but not encrypted
